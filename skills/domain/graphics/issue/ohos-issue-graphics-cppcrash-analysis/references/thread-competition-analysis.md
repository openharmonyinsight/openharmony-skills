# 多线程竞争分析

## 分析其它线程状态

### 1. 查看其它线程的调用栈

从`Other thread info`部分，查看所有线程的调用栈。

重点关注：

- 是否有其它线程也在访问相同的数据结构
- 是否有其它线程持有相同的锁
- 是否有其它线程在修改相同的对象

### 2. 识别锁的竞争

检查是否有多个线程在竞争同一个锁。

示例：

```
RSSubThread2（崩溃线程）：正在遍历 RenderNodeDrawableCache_
RSUniRenderThread: 同时持有 cacheMapMutex_ 锁，调用 GetDrawableById
```

### 3. 分析线程安全问题

检查代码中的线程安全机制：

- 是否正确使用了互斥锁
- 锁的范围是否足够
- 是否存在死锁风险
- 迭代器在遍历过程中是否会失效

### 4. 识别竞态条件

分析可能存在的竞态条件：

- 一个线程在读取，另一个线程在写入
- 一个线程在遍历容器，另一个线程在修改容器
- 对象的生命周期在多线程间没有正确同步
