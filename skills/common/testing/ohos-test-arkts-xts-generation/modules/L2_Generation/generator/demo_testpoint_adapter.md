# 设计文档 → Demo 测试点转换适配器

> **模块信息**
> - 层级：L2_Generation
> - 优先级：按需加载
> - 适用范围：Phase 5 Step 1（Demo 生成）前的测试点转换
> - 依赖：Phase 4 设计文档

---

## 一、模块概述

将 Phase 4 生成的设计文档（`.design.md`）中 UI 类测试用例转换为 demo-pipeline 可消费的 `demo_test_points.md` 格式。

---

## 二、转换映射表

| .design.md 字段 | demo_test_points.md 字段 | 转换规则 |
|-----------------|------------------------|---------|
| 用例编号 | 测试点ID | 直接映射 |
| 用例名 | 描述 | 直接映射 |
| 类型 | 技术类型 | PARAM/ERROR/RETURN/BOUNDARY/EVENT 直接映射 |
| 级别 | 优先级 | P0/P1/P2 直接映射 |
| 测试步骤 | 操作步骤 | 代码步骤 → 用户操作描述 |
| 预期结果 | 预期结果 | 代码断言 → 可观察 UI 行为 |
| 控件操作序列 | 控件操作序列 | 直接映射（Phase 4 已定义） |
| 页面 | 页面 | 直接映射（Phase 4 已定义） |

---

## 三、操作步骤转换规则

### 3.1 代码步骤 → 用户操作描述

| 原始代码步骤 | 转换后的用户操作 |
|-------------|----------------|
| `调用 methodA(param1="value")` | `在输入框输入参数值，点击执行按钮` |
| `验证 result === expected` | `查看结果区域显示预期值` |
| `验证 error.code === 401` | `查看结果区域显示错误码` |
| `调用 component.setWidth(200)` | `在宽度输入框输入 200，点击执行按钮` |
| `监听 onClick 事件` | `点击目标组件，查看事件触发状态` |

### 3.2 预期结果转换规则

| 原始代码断言 | 转换后的 UI 预期 |
|-------------|----------------|
| `expect(result).assertEqual(expected)` | `结果区域显示 {expected}` |
| `expect(error.code).assertEqual(401)` | `结果区域显示错误码 401，状态为 FAIL` |
| `expect(visible).assertEqual('true')` | `组件可见，状态为 PASS` |
| `expect(text).assertEqual('16.00fp')` | `结果区域显示 '16.00fp'，状态为 PASS` |

## 四、输出格式

转换后的 `demo_test_points.md` 格式：

```markdown
# Demo 测试点

## 测试点列表

| 测试点ID | 描述 | 技术类型 | 优先级 | 页面 | 操作步骤 | 预期结果 |
|---------|------|---------|--------|------|---------|---------|
| SUB_ARKUI_XXX_EVENT_001 | 组件属性正常值测试 | EVENT | P0 | PAGE-001 | 在输入框输入宽度值 200，点击执行按钮 | 结果区域显示 '200.00vp'，状态 PASS |
| SUB_ARKUI_XXX_EVENT_002 | 组件属性空值测试 | EVENT | P1 | PAGE-001 | 不输入参数，点击执行按钮 | 结果区域显示错误码，状态 FAIL |
```

---

## 五、转换约束

1. **仅转换 UI 类用例**：非 UI 类用例（纯 API 调用）不在此转换范围内
2. **保留设计文档关联**：测试点ID 必须与设计文档的用例编号一致
3. **控件 ID 引用**：操作步骤中引用的控件 ID 必须在设计文档的控件 ID 清单附录中存在
4. **页面引用**：页面 ID 必须在设计文档的页面规划中存在
