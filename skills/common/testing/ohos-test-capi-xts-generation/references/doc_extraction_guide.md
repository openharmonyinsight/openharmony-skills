# 文档提取指南

> **模块信息**
> - 类型：参考文档
> - 用途：指导从各类文档中提取 XTS 测试用例生成所需的信息

---

## 一、API 文档提取

### 1.1 需要提取的信息

从 API 文档中提取以下信息用于测试用例生成：

| 提取项 | 说明 | 用途 |
|--------|------|------|
| 函数签名 | 返回类型 + 函数名 + 参数列表 | 确定测试函数名和参数 |
| 参数说明 | 参数名、类型、必填/可选、取值范围 | 生成参数测试和边界值测试 |
| 返回值说明 | 返回值类型、可能的返回值 | 生成返回值测试 |
| 错误码 | 错误码值、触发条件、错误信息 | 生成错误码测试 |
| 版本信息 | 起始支持版本、废弃版本 | 确定测试兼容性 |
| 系统能力 | SystemCapability 标识 | 配置 syscap.json |
| 权限要求 | 所需权限列表 | 配置测试环境 |
| 注意事项 | 线程安全、内存管理等特殊说明 | 生成特殊测试场景 |

### 1.2 提取结果格式

提取完成后，应整理为以下 JSON 结构：

```json
{
  "api_name": "函数名",
  "signature": {
    "return_type": "返回类型",
    "params": [
      {"name": "参数名", "type": "类型", "required": true, "description": "说明"}
    ]
  },
  "return_values": [
    {"value": "返回值", "description": "说明"}
  ],
  "error_codes": [
    {"code": "错误码", "trigger": "触发条件", "message": "错误信息"}
  ],
  "syscap": "SystemCapability.XXX",
  "permissions": ["权限列表"],
  "since_version": "起始版本",
  "notes": ["注意事项"]
}
```

## 二、测试设计文档提取

### 2.1 需要提取的信息

从测试设计文档中提取：

| 提取项 | 说明 | 用途 |
|--------|------|------|
| 子系统/模块 | 子系统和模块名称 | 命名规范 SUB_[子系统]_[模块] |
| 测试用例编号 | SUB_XXX_XXX_NNN 格式 | 确保命名一致性 |
| 测试类型 | PARAM/ERROR/RETURN/BOUNDARY/MEMORY | 确定测试模板 |
| 测试步骤 | 具体操作步骤 | 生成测试代码 |
| 预期结果 | 期望的测试结果 | 生成断言 |
| 前置条件 | 测试执行的前提 | 配置测试环境 |
| 覆盖率统计 | 已覆盖/未覆盖的测试场景 | 分析测试完整性 |

### 2.2 测试类型判定规则

根据文档描述自动判定测试类型：

| 文档关键词 | 测试类型 |
|-----------|---------|
| 正常值、正常参数、有效输入 | PARAM |
| 错误码、异常、无效参数 | ERROR |
| 返回值、返回结果 | RETURN |
| 边界值、最大值、最小值、临界 | BOUNDARY |
| 内存分配、内存释放、内存泄漏 | MEMORY |

## 三、头文件 (.h) 提取

### 3.1 需要提取的信息

从 C 头文件中提取：

| 提取项 | 说明 | 提取方法 |
|--------|------|---------|
| 函数声明 | 返回类型 + 函数名 + 参数列表 | 正则匹配函数声明模式 |
| 宏定义 | 常量、枚举值 | 匹配 `#define`、`enum` |
| 类型定义 | 结构体、枚举、typedef | 匹配 `struct`、`enum`、`typedef` |
| 包含依赖 | 依赖的头文件 | 匹配 `#include` |
| 条件编译 | 平台/版本条件 | 匹配 `#ifdef`、`#if` |

### 3.2 函数签名解析规则

解析 C 函数声明时的关键信息：

- **返回类型**：void、int、char*、结构体指针等
- **函数名**：遵循子系统命名规范
- **参数列表**：每个参数的类型和名称
- **修饰符**：const、static、EXTERN_C 等

## 四、配置文档提取

### 4.1 BUILD.gn 提取

| 提取项 | 说明 |
|--------|------|
| 测试套类型 | ohos_js_app_suite / ohos_app_assist_suite |
| hap_name | HAP 文件名 |
| part_name | 部件名 |
| subsystem_name | 子系统名 |
| deps | 依赖列表 |

### 4.2 Test.json 提取

| 提取项 | 说明 |
|--------|------|
| bundle-name | 测试包名 |
| module-name | 模块名 |
| test-timeout | 超时时间 |
| test-file-name | HAP 文件名 |

### 4.3 oh-package.json5 提取

| 提取项 | 说明 |
|--------|------|
| name | 模块名 |
| devDependencies | 开发依赖 |
| overrides | 依赖覆盖 |

## 五、文档格式支持

支持的文档格式和提取策略：

| 格式 | 提取策略 |
|------|---------|
| Markdown (.md) | 按标题层级提取，解析表格和代码块 |
| HTML (.html/.htm) | 按标签结构提取，支持 API 文档页面 |
| JSON (.json) | 直接解析 JSON 结构 |
| C 头文件 (.h) | 正则匹配函数声明和类型定义 |
| JSON5 (.json5) | 去除注释后解析为 JSON |

## 六、提取信息质量检查

完成提取后，检查以下内容：

- [ ] 所有 API 函数签名都已提取
- [ ] 参数类型和返回类型完整
- [ ] 错误码列表完整
- [ ] 系统能力标识正确
- [ ] 版本信息已记录
- [ ] 依赖关系已梳理
