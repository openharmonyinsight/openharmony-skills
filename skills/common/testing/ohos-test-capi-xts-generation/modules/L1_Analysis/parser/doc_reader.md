## 模块概述

文档阅读模块负责从各类文档中提取 XTS 测试用例生成所需的信息。

### 支持的文档类型

| 文档类型 | 格式 | 提取内容 |
|---------|------|---------|
| API 文档 | .md / .html | 函数签名、参数说明、返回值、错误码 |
| 头文件 | .h | 函数声明、宏定义、类型定义 |
| 测试设计文档 | .md | 测试用例、测试场景、预期结果 |
| 配置文档 | .json / .json5 / .gn | 配置项、依赖项、环境要求 |

### 提取信息概要

从 API 文档和头文件中提取以下核心信息：

1. **函数签名** - 返回类型、函数名、参数列表（类型+名称）
2. **参数说明** - 每个参数的取值范围、必填/可选、特殊约束
3. **返回值** - 可能的返回值及其含义
4. **错误码** - 错误码值、触发条件、错误信息
5. **系统能力** - SystemCapability 标识
6. **权限要求** - 所需权限列表
7. **版本信息** - 起始支持版本、废弃版本

### 提取结果格式

提取的信息应整理为结构化 JSON，包含：
- `api_name`: API 名称
- `signature`: 函数签名（返回类型 + 参数列表）
- `return_values`: 返回值说明列表
- `error_codes`: 错误码列表
- `syscap`: 系统能力标识
- `permissions`: 权限要求
- `since_version`: 起始版本

### 测试类型判定

根据提取的信息自动判定应生成的测试类型：

| 条件 | 测试类型 |
|------|---------|
| 有正常参数 | PARAM |
| 有错误码定义 | ERROR |
| 有明确返回值 | RETURN |
| 有数值/字符串参数 | BOUNDARY |
| 涉及内存分配 | MEMORY |
