# Evals for ohos-dev-sa-codegen

This directory contains test cases for the `ohos-dev-sa-codegen` skill.

## Test Cases

### 1. generate-idl-sa-from-scratch

Tests the skill's ability to generate a complete IDL 化 SA framework from user-provided parameters:
- Generates all 9 required files with correct variable substitution
- IDL interface with correct method signatures (in/out annotations)
- SA Profile JSON with matching SA ID
- CFG uses `/system/bin/sa_main` and correct ondemand flag
- services/BUILD.gn includes `idl_gen_interface`, `shlib_type = "sa"`
- REGISTER_SYSTEM_ABILITY_BY_ID macro with correct parameters
- No leftover template placeholder values

**Key assertions**: generates-all-9-files, idl-file-correct, sa-id-3901-in-profile, cfg-uses-sa-main, cfg-ondemand-true, services-build-has-idl-gen, services-build-has-shlib-type, register-macro-correct, no-placeholder-values

### 2. generate-idl-sa-incomplete-info

Tests the skill's ability to **refuse code generation and collect missing fields** when user provides incomplete information:
- Only provides SA class name and SA ID (2 of 10 required fields)
- Should NOT generate any code
- Should ask for all missing fields (process name, subsystem, part, startup mode, distributed, HiLog domain, interface methods, output path)
- Should NOT use default values or placeholders

**Key assertions**: does-not-generate-code, asks-for-process-name, asks-for-subsystem, asks-for-log-domain, asks-for-interface-methods, no-default-values

### 3. modify-existing-sa-add-interface

Tests the skill's ability to guide adding a new interface method to an existing IDL 化 SA:
- Correct IDL method signature with in/out annotations
- Override implementation needed in SA class
- Proxy/Stub auto-regenerated by IDL tool
- Reminds user to recompile

**Key assertions**: idl-method-signature-correct, mentions-override-impl, auto-regen-proxy-stub, reminds-recompile

### 4. migrate-nonidl-to-idl

Tests the skill's ability to guide migrating an existing 手写 Proxy/Stub SA to IDL 模式:
- Reads existing code first before suggesting changes
- Generates correct IDL file with type mappings
- Modifies BUILD.gn correctly (add idl_gen_interface, remove hand-written sources)
- Lists files to delete
- Shows C++ → IDL type mapping table

**Key assertions**: reads-existing-code-first, generates-idl-file, modifies-build-gn, lists-files-to-delete, type-mapping-table

## Design Philosophy

These test cases cover the 3 main modes of the skill and verify behaviors that are **unique to the skill**:

| Behavior | With Skill | Without Skill (baseline) |
|----------|-----------|------------------------|
| Complete file generation (9 files) | Generates all required files with correct variable substitution | Incomplete or uses template placeholder values |
| Information validation | Refuses to generate with incomplete info, collects all 10 fields | Generates code with placeholder/default values |
| IDL syntax correctness | Correct in/out annotations, type mappings, interface declaration | Incorrect IDL syntax or missing annotations |
| CFG correctness | Uses `/system/bin/sa_main`, correct ondemand flag | Uses wrong binary path or missing ondemand |
| shlib_type | Includes `shlib_type = "sa"` in BUILD.gn | Missing symbol visibility control |
| Migration completeness | Step-by-step: analyze → generate → modify → delete → explain | Skips steps or doesn't list files to delete |
