# 高频文档错误模式库

## 模式 1：参数作用互换

**错误特征：** 两个参数的描述内容互换了位置。

**典型场景：**
- SVG `matrix(a,b,c,d,e,f)` 中 b（skewY）和 c（skewX）的描述写反
- CSS `transform: translate(x, y)` 中 x/y 方向描述互换
- 渐变角度中起始方向和结束方向互换

**检查方法：**
1. 查标准定义中每个参数的数学含义
2. 用标准示例中的具体数值验证：代入参数后，效果是否与文档描述一致
3. 如果只有两个参数且描述对称，重点检查互换

**验证示例（SVG matrix）：**

标准定义（W3C SVG）：
```
[a  c  e]   a = scaleX,  b = skewY
[b  d  f]   c = skewX,   d = scaleY
[0  0  1]   e = translateX, f = translateY
```

错误文档：b 控制 x 方向倾斜，c 控制 y 方向倾斜
正确文档：b 控制 y 方向倾斜（skewY），c 控制 x 方向倾斜（skewX）

## 模式 2：方向描述错误

**错误特征：** x/y、水平/垂直、顺时针/逆时针等方向性描述搞反。

**典型场景：**
- border-radius 四个角的顺序描述错误
- 动画方向（顺时针 vs 逆时针）搞反
- 渐变方向角度的起始方向描述错误

**检查方法：**
1. 查标准中方向性的精确定义（通常用数学坐标或角度定义）
2. 注意坐标系差异：屏幕坐标系 y 轴向下，数学坐标系 y 轴向上

## 模式 3：参数顺序错误

**错误特征：** 参数位置与标准函数签名不一致。

**典型场景：**
- `rgba(r, g, b, a)` 参数顺序写错
- `rect(x, y, width, height)` 顺序描述与标准不同
- 函数重载的参数列表顺序与标准不同

**检查方法：**
1. 直接对比标准函数签名的参数顺序
2. 注意某些标准有位置参数和命名参数的区别

## 模式 4：类型与值域错误

**错误特征：** 参数的取值范围或数据类型描述与标准不符。

**典型场景：**
- 百分比值写成像素值
- 角度范围 [0, 360] 写成 [0, 2π]
- 整数类型写成浮点类型
- 默认值与标准定义不一致

**检查方法：**
1. 查标准中的类型定义（Type: <number> | <percentage> | <length>）
2. 查标准中显式声明的约束条件
3. 注意隐式约束：某些参数数学上不能为负，但标准可能未显式说明

## 模式 5：枚举值遗漏或错误

**错误特征：** 枚举类型的可选值不完整或写错。

**典型场景：**
- `display` 枚举值遗漏了某些选项
- `visibility` 枚举值名称拼写错误
- 版本升级后新增的枚举值未同步到文档

**检查方法：**
1. 对比标准中完整的枚举定义
2. 注意标准版本差异：新版本可能新增枚举值
3. 注意平台扩展值：OpenHarmony 可能在标准基础上扩展了枚举

## 模式 6：数学公式/算法描述错误

**错误特征：** 文档中的公式或算法描述与标准数学定义不一致。

**典型场景：**
- 插值算法描述错误（线性 vs 缓动）
- 贝塞尔曲线参数含义描述错误
- 颜色空间转换公式错误

**检查方法：**
1. 用标准中给出的具体示例值代入验证
2. 对比标准附录中的数学推导
3. 用边界值测试公式是否正确

## 通用自查清单

验证每条声明时，逐项确认：

- [ ] 是否找到了对应的权威标准文档？
- [ ] 标准版本是否与文档描述的功能版本匹配？
- [ ] 参数顺序是否与标准签名一致？
- [ ] 每个参数的数学含义是否与标准一致？
- [ ] 类型、值域、默认值是否与标准一致？
- [ ] 枚举值是否完整？
- [ ] 方向性描述是否正确（坐标系方向）？
- [ ] 是否有平台扩展或限制需要特别说明？
