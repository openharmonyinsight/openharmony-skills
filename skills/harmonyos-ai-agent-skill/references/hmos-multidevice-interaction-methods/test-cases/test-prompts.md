# Interaction Methods Test Prompts

用于验证 `interaction-methods` 的三层测试能力。

## 1. Smoke Prompt

桌面端页面需要同时支持鼠标 hover、右键菜单和键盘焦点导航，并确保触摸和鼠标的交互一致。请给出开发阶段方案。

## 2. Scene-Functional Prompt

请为桌面端内容页增加鼠标 hover、右键菜单、键盘方向键导航和快捷键，并保证触摸设备仍能正常工作。请输出可落到工程中的开发方案。

## 3. Real-World Prompt

桌面端富文本编辑器上线后收到多起反馈：鼠标悬浮工具栏按钮没有反馈，Tab 和方向键切焦点顺序混乱，右键菜单和快捷键还会互相抢焦点。请输出修复方案和工程验证要求，同时确保触摸平板模式不受影响。

## 典型验证点

- 是否命中 `INPUT-03`
- 是否将输入方式变化视为主问题
- 是否保持布局问题与交互问题的边界
- 是否给出桌面输入回归矩阵
