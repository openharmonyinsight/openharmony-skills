# Hardware Access Test Prompts

用于验证 `hardware-access` 的三层测试能力，并兼容远端已合并的相机场景口径。

## 1. Smoke Prompts

### 能力检测与降级

应用需要根据设备是否支持蓝牙外设和摄像头能力做差异化接入，并在能力不足时提供降级路径。请给出开发阶段建议。

### 相机适配

折叠屏相机应用在折叠/展开切换后出现黑屏，同时预览画面方向与设备持握方向不一致。展开后拍照的照片方向也发生了 90° 旋转。请给出问题修复思路。

## 2. Scene-Functional Prompt

请为需要蓝牙外设接入和摄像头能力的功能补齐能力检测与降级方案，要求不支持能力的设备能给出替代路径。请输出可落到工程中的开发方案。

## 3. Real-World Prompt

智能设备配网页面最近出现多起机型问题：部分平板没有后置摄像头，扫码入口直接报错；部分设备外接蓝牙设备配网按钮点击无反应。产品要求保留一套页面，但在能力不足时给出替代路径。请输出修复方案和工程验证要求。

## 典型验证点

### 能力检测（HW-01）

- 是否命中 `HW-01`
- 是否先判断能力，再讨论接入
- 是否包含降级和兼容分支

### 相机适配（HW-02）

- 是否命中 `HW-02`
- 是否识别折叠态相机重建是主问题
- 是否联动预览旋转和拍照旋转作为次级场景
- 是否区分根因（相机设备不可用）和场景现象（黑屏、旋转异常）

### 三层补充要求

- `scene-functional` / `real-world` 最终仍按测试用例结果判断
- 是否定义真机能力矩阵和降级验证要求
