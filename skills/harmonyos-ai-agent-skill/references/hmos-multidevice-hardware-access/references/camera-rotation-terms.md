# 相机旋转角度术语基线（官方基线）

## 目标

方向类问题最容易先在角度概念层出错。本文档定义设备自然方向、屏幕旋转角度、镜头安装角度、预览旋转角度等核心术语，作为旋转适配的概念基线。

来源：[华为开发者文档 - 相机旋转角度术语](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/camera-rotation)

## 核心术语

### 设备自然方向

设备默认的使用方向。

- **手机**：竖屏且充电口向下
- **平板**：横屏

### 屏幕旋转角度

显示设备的屏幕顺时针旋转角度，即设备从自然方向到当前方向的顺时针夹角。

通过 `Display.rotation` 获取，枚举值与角度的对应关系：
- `Display.rotation = 0` → 0°
- `Display.rotation = 1` → 90°
- `Display.rotation = 2` → 180°
- `Display.rotation = 3` → 270°

### 相机镜头安装角度

相机采集图像方向到设备自然方向在顺时针方向的夹角。

- **后置相机**：手机后置相机传感器是横屏安装的，竖屏使用时原始图像需顺时针旋转 90° 才能与设备自然方向一致。**后置镜头角度 = 90°**
- **前置相机**：前置镜头朝向使用者，出图方向与后置出图方向互为镜像。**前置镜头角度 = 270°**

### 预览旋转角度

> 推荐通过接口 `getPreviewRotation` 获取，不需要手动计算。

**计算公式**：`预览旋转角度 = 镜头安装角度 + 屏幕旋转角度`

#### 后置相机预览角度示例

| 设备条件 | 镜头角度 | 屏幕旋转角度 | 预览旋转角度 |
|---------|---------|------------|------------|
| 竖屏充电口向下 | 90° | 0° | 90° + 0° = 90° |
| 横屏充电口向左 | 90° | 90° | 90° + 90° = 180° |
| 竖屏充电口向上 | 90° | 180° | 90° + 180° = 270° |
| 横屏充电口向右 | 90° | 270° | 90° + 270° = 0° |

#### 前置相机预览角度示例

前置相机有镜像出图特性。

| 设备条件 | 镜头角度 | 屏幕旋转角度 | 预览旋转角度 |
|---------|---------|------------|------------|
| 竖屏充电口向下 | 270° | 0° | 270° + 0° = 270° |
| 横屏充电口向左 | 270° | 90° | 270° + 90° = 0° |
| 竖屏充电口向上 | 270° | 180° | 270° + 180° = 90° |
| 横屏充电口向右 | 270° | 270° | 270° + 270° = 180° |

### 拍照/录像旋转角度

在拍照、录像时，图像旋转角度与设备重力方向（即设备旋转角度）相关。

- **使用后置相机拍摄**：`图像旋转角度 = 镜头安装角度 + 重力方向`
- **使用前置相机拍摄**：`图像旋转角度 = 镜头安装角度 - 重力方向`

应用需要监听 `SensorId.GRAVITY` 事件，获取重力传感器 x、y、z 数据来计算设备旋转角度。

## 术语速查表

| 术语 | 定义 | 获取方式 |
|------|------|---------|
| 设备自然方向 | 设备默认使用方向 | 设备固有属性 |
| 屏幕旋转角度 | 从自然方向到当前方向的顺时针夹角 | `Display.rotation`（枚举值 ×90°） |
| 相机镜头安装角度 | 采集图像方向到自然方向的顺时针夹角 | 后置 90°，前置 270° |
| 预览旋转角度 | 镜头安装角度 + 屏幕旋转角度 | `previewOutput.getPreviewRotation()` |
| 拍照旋转角度 | 后置：镜头安装角度 + 重力方向；前置：镜头安装角度 - 重力方向 | 手动计算，设入 `capture()` 的 `rotation` 参数 |

## 使用场景

| 场景 | 触发条件 | 推荐用法 | 为什么要判断 | 不判断的风险 |
| --- | --- | --- | --- | --- |
| 预览旋转概念理解 | 需要理解为什么预览方向不对 | 查阅"预览旋转角度"公式 | 角度概念混淆是旋转问题的常见根因。 | 盲目调参，越调越乱。 |
| 拍照方向异常排查 | 照片方向与预期不符 | 查阅"拍照旋转角度"公式 + 前后置区分 | 前置用减法，后置用加法，混淆即出错。 | 修复后仍然方向错误。 |
| Surface 比例异常 | 画面拉伸/压缩 | 查阅屏幕旋转角度与 Surface 比例关系 | 0°/180° 和 90°/270° 规则不同。 | 比例始终对不上。 |
