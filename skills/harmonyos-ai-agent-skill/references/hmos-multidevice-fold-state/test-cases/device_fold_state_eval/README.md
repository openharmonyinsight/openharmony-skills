# Device Fold State Eval Cases

用于在 `references/hmos-multidevice-fold-state` 下直接复用的评测用例集。

## 目录

- `hover_state_eval/`：悬停态分屏与方向稳定
- `crease_avoidance_eval/`：折痕定位与避让边界
- `fold_continuity_eval/`：开合连续性恢复链路

## 用途

- 评估折叠场景在开发方案、问题修复和验证计划三个阶段的输出质量。
- 支持从“策略描述”到“可执行验证项”的闭环检查。

## 验证点

- 场景路由是否准确：主场景与次场景是否符合问题语义。
- 修复策略是否完整：监听入口、方向决策、分区职责、生命周期回收是否覆盖。
- 验证计划是否可执行：是否要求状态、日志、截图、布局树等证据组合。

## 关键词

- 悬停态：`HALF_FOLD`、上展示下交互、`hover enabled/disabled`、退出恢复。
- 折痕避让：同坐标系映射、折痕边界锚定、`16vp/40vp` 安全间距。
- 开合连续性：状态快照、双触发恢复、暂停态优先、时间轴连续。
