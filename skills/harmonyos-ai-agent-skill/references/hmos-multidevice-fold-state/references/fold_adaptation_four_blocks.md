# 折展适配三块主线（结构化总览）

本文档用于统一 `device-fold-state` 的折展适配结构，按三块主线组织：
- 悬停适配
- 折痕避让
- 开合连续性

## 官方来源

- 华为开发者联盟《折叠屏应用 UX 体验标准》
- 链接：https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-foldable-screen-0000001807866557
- 对齐条目：4.1（开合连续性）、4.3（悬停适配）、4.4（折痕避让）

## 三块主线映射

| 主线 | 官方条目 | 主路径文档 | 关键验收点 |
| --- | --- | --- | --- |
| 悬停适配 | 4.3 | `./hover_state_interaction.md` | 上半屏展示、下半屏交互，悬停态职责稳定 |
| 折痕避让 | 4.4 | `./crease_avoidance.md` | 上半屏避让 16 vp，下半屏避让 40 vp |
| 开合连续性 | 4.1 | `./fold_continuity.md` | 无步骤增加；滚动/输入/清晰度/播放连续 |

## 使用建议

1. 新需求评审：先按三块主线判断必做范围。
2. 开发实现：优先走每块的 P0 主路径，再补 P1 工程模板。
3. 问题定位：先判定问题属于哪一块，避免跨块误修。
4. 验证阶段：按三块分别给证据，避免只验证单点功能。
