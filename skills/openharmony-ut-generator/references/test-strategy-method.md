# 测试策略和测试方法参考文档

## 概述

本文档介绍如何制定测试策略和进行测试设计，为 OpenHarmony 单元测试提供方法论指导。

## 测试策略制定

### 什么是测试策略

测试策略是测试工作的总体规划和指导方针，包括：
- 测试范围
- 测试类型
- 测试优先级
- 覆盖率目标
- 测试方法选择

### 测试策略制定步骤

```
1. 分析被测对象
   └────────────────────────────────────┐
   ↓                                     │
2. 确定测试范围                          │
   └────────────────────────────────────┤
   ↓                                     │
3. 选择测试类型                          │
   └────────────────────────────────────┤
   ↓                                     │
4. 划分优先级                            │
   └────────────────────────────────────┤
   ↓                                     │
5. 设定覆盖率目标                        │
   └────────────────────────────────────┤
   ↓                                     │
6. 输出策略文档                          │
   └────────────────────────────────────┘
```

### 1. 分析被测对象

分析源码，识别：

| 分析维度 | 内容 |
|----------|------|
| 公开接口 | 类的 public 方法、模块的导出函数 |
| 参数特征 | 参数类型、取值范围、必填/可选 |
| 返回值 | 返回类型、成功/失败标识、错误码 |
| 异常处理 | 异常类型、错误处理路径 |
| 依赖关系 | 依赖的类/模块、外部服务 |
| 状态管理 | 状态变量、状态转换 |

### 2. 确定测试范围

**原则**：
- 公开 API 必须测试
- 核心功能优先
- 复杂逻辑重点覆盖

**范围划分示例**：

```markdown
### 测试范围

#### 必测项（P0）
- Calculator::Add/Sub - 核心计算功能
- Calculator::Initialize - 初始化功能

#### 重点测试（P1）
- Calculator::Multiply/Divide - 扩展计算功能
- Calculator::ValidateInput - 输入验证

#### 一般测试（P2）
- Calculator::GetStatus - 状态查询
- Calculator::Log - 日志记录

#### 可选测试（P3）
- Calculator::GetVersion - 版本信息
```

### 3. 选择测试类型

根据被测对象特性选择测试类型：

| 测试类型 | 适用场景 | 编码 |
|----------|----------|------|
| 功能测试 | 验证功能正确性 | FUNC |
| 性能测试 | 验证性能指标 | PERF |
| 可靠性测试 | 验证稳定性 | RELI |
| 安全测试 | 验证安全性 | SECU |
| 模糊测试 | 安全性/稳定性 | FUZZ |

**示例**：

```markdown
### 测试类型

#### 功能测试（FUNC）
- 所有公开 API 的正常功能验证
- 输入输出正确性验证
- 状态转换验证

#### 性能测试（PERF）
- 高频调用场景性能验证
- 大数据量处理性能

#### 安全测试（SECU）
- 输入边界安全验证
- 权限检查验证
```

### 4. 划分优先级

**优先级划分原则**：

| 优先级 | 判断依据 |
|--------|----------|
| P0 | 功能失败导致系统不可用 |
| P1 | 功能失败导致核心功能异常 |
| P2 | 功能失败导致用户体验下降 |
| P3 | 功能失败影响较小 |

**与测试等级对应**：

| 优先级 | 测试等级 |
|--------|----------|
| P0 | Level0 |
| P1 | Level1 |
| P2 | Level2 |
| P3 | Level3/Level4 |

### 5. 设定覆盖率目标

**覆盖率类型**：

| 覆盖率类型 | 说明 | 目标建议 |
|------------|------|----------|
| 语句覆盖 | 每条语句至少执行一次 | ≥80% |
| 分支覆盖 | 每个分支至少执行一次 | ≥60% |
| 函数覆盖 | 每个函数至少调用一次 | ≥90% |

**目标设定示例**：

```markdown
### 覆盖率目标

- 语句覆盖率：≥80%
- 分支覆盖率：≥60%
- 函数覆盖率：100%（所有公开函数）
```

### 测试策略输出模板

```markdown

```

## 测试设计方法

### 什么是测试设计

测试设计是将测试策略转化为具体测试场景的过程，包括：
- 测试场景设计
- 测试输入设计
- 预期结果定义
- 测试用例规划

### 测试设计步骤

```
1. 确定测试对象（函数/类）
         ↓
2. 分析功能逻辑
         ↓
3. 划分测试场景
         ↓
4. 设计输入数据
         ↓
5. 定义预期结果
         ↓
6. 输出测试设计表
```

### 测试方法分类

测试方法分为两大类，在单元测试中各有适用场景：

```
测试方法
├── 白盒测试（基于代码结构）
│   ├── 语句覆盖 (Statement Coverage)
│   ├── 分支/判定覆盖 (Branch/Decision Coverage)
│   ├── 条件覆盖 (Condition Coverage)
│   ├── 判定-条件覆盖 (Decision-Condition Coverage)
│   ├── 条件组合覆盖 (Multiple Condition Coverage)
│   ├── MC/DC 覆盖 (Modified Condition/Decision Coverage)
│   └── 路径覆盖 (Path Coverage)
│
└── 黑盒测试（基于功能规格）
    ├── 等价类划分 (Equivalence Partitioning)
    ├── 边界值分析 (Boundary Value Analysis)
    ├── 判定表 (Decision Table)
    ├── 因果图 (Cause-Effect Graph)
    ├── 正交实验法 (Orthogonal Experimental)
    ├── 状态迁移测试 (State Transition Testing)
    └── 错误推测法 (Error Guessing)
```

#### 白盒与黑盒方法选择指南

| 被测对象特征 | 推荐方法 | 说明 |
|-------------|----------|------|
| 复杂条件判断逻辑 | 白盒：MC/DC、条件组合覆盖 | 确保每个条件的影响都被验证 |
| 多分支 if/switch 逻辑 | 白盒：分支覆盖 + 黑盒：等价类划分 | 确保每个分支被走到，每类输入被覆盖 |
| 输入参数取值范围明确 | 黑盒：边界值分析 + 等价类划分 | 高效覆盖典型和边界输入 |
| 状态机/多状态逻辑 | 黑盒：状态迁移测试 | 系统化覆盖状态转换 |
| 多输入组合关系 | 黑盒：判定表 / 正交实验法 | 系统化覆盖组合场景 |
| 经验丰富的已知缺陷模式 | 黑盒：错误推测法 | 针对性补充易错点 |

### 测试场景划分

#### 基本场景类型

| 场景类型 | 说明 | 示例 |
|----------|------|------|
| 正常路径 | 正常输入，正常输出 | Add(10, 5) → 15 |
| 异常路径 | 异常输入，错误处理 | Divide(10, 0) → 错误 |
| 边界条件 | 边界值输入 | Add(INT_MAX, 1) |
| 状态相关 | 特定状态下的行为 | 未初始化时调用 |
| 并发场景 | 多线程/并发情况 | 多线程同时调用 |

#### 场景划分方法

**等价类划分**：

| 输入类型 | 有效等价类 | 无效等价类 |
|----------|------------|------------|
| 整数参数 | 正数、负数、零 | 超范围值 |
| 字符串参数 | 非空字符串 | 空字符串、null |
| 对象参数 | 有效对象 | null、无效对象 |

**边界值分析**：

| 边界类型 | 测试点 |
|----------|--------|
| 数值边界 | 最小值、最大值、边界±1 |
| 长度边界 | 0、最大长度、最大长度±1 |
| 索引边界 | 0、数组长度-1、数组长度 |

### 输入数据设计

#### 正常输入

- 典型业务场景的输入值
- 合法参数组合
- 常见使用模式

#### 异常输入

- 空值（null/nullptr）
- 空字符串
- 零值
- 超范围值
- 无效参数组合

#### 边界输入

- 最小边界值
- 最大边界值
- 刚超边界值

### 预期结果定义

| 结果类型 | 定义方式 |
|----------|----------|
| 返回值 | 预期返回的具体值 |
| 状态变化 | 预期的状态改变 |
| 异常抛出 | 预期抛出的异常类型 |
| 错误码 | 预期返回的错误码 |

```markdown

| Add | T01 | 正常加法 | (10, 5) | 15 | 正常 |

| Divide | T05 | 除零异常 | (10, 0) | -1 | 异常 |
```

## 测试策略与设计的关系

```
测试策略（宏观规划）
     │
     ├─ 测试范围 → 确定哪些函数需要测试
     ├─ 测试类型 → 确定测试设计类型选择
     ├─ 优先级 → 确定测试设计等级分配
     ├─ 覆盖率 → 确定测试设计场景数量
     │
     ↓
测试设计（具体场景）
     │
     ├─ 测试场景 → 具体测试情况
     ├─ 输入数据 → 具体参数值
     ├─ 预期结果 → 具体验证值
     │
     ↓
测试用例（实现代码）
```
