# cangjie-vec-retriever-guide

## 目的

当基础仓颉 Skills（例如 `cangjie-kernel` 下的各个子技能）无法命中知识点、答案不准确/过时/不适用，或工程出现编译错误、构建失败、运行异常、报错信息难以理解时，由本 Skill 先行接管，通过 **语义向量 RAG 检索（L1 阶段）** 提供更高召回率的仓颉知识点与代码示例。

> 本 Skill 只负责 **L1 语义检索阶段**：调用 Python 检索脚本，从向量库 + BM25 混合检索中返回候选答案；当语义检索结果不足以解决问题时，再由 `cangjie-doc-search-guide` 进入 L3 官方文档“考古”阶段。

## 语义检索脚本与路径

- **L1 RAG 查询脚本**（位于 `skills/utils/scripts/ask_cangjie.py`）：
  - 通过如下命令执行，用于向量 RAG + BM25 混合检索仓颉知识库：

```bash
cd skills/utils/scripts
python ask_cangjie.py "<待查询的技术关键词>"
```

- **经验记录文档**（可选联动）：`skills/cangjie-evolution/Evolution.md` 用于记录本次问题和最终解决方案，便于后续复用。

## L1：语义 RAG 知识检索流程

### 触发时机

满足以下任一情况时，应调用本 Skill：

- 已尝试 `cangjie-kernel` 中相关技能，但：
  - 命中的知识点明显不完整，或
  - 回答明显过时，或
  - 示例代码与当前工程场景不匹配；
- 当前工程出现：
  - 复杂的编译错误，报错栈难以从基础技能中直接找到解释；
  - 构建失败且错误信息涉及多组件/多模块；
  - 运行期异常信息含有大量英文或内部类名，难以理解；
- 用户提出的需求涉及多个概念组合，基础 L1 技能难以涵盖。

### 强制调用规范

你必须**显式**调用终端/命令行工具，执行如下命令格式之一：

```bash
python skills/utils/scripts/ask_cangjie.py "<核心关键词>"
```

或在已 `cd` 至脚本目录后：

```bash
python ask_cangjie.py "<核心关键词>"
```

### 关键词选择策略

- **优先使用纯英文 API / 模块名**：
  - 例如：`std.fs`, `stdx.net.http`, `HashMap`, `JsonValue`, `WebSocket`；
  - 不要附带中文描述（如 “Http 客户端怎么写”），以免影响检索精度。
- **尽量精确到一个核心技术点**：
  - 每次调用只检索一个主关键词，例如：
    - `HTTP client`
    - `JsonValue`
    - `std.fs`
  - 如果一个任务涉及 “文件读写 + JSON 解析 + HTTP 请求”，应分三次调用脚本，分别使用对应的英文关键词。
- **中英文混合策略**：
  - 若使用英文关键词得到的结果明显不相关或覆盖不足，可以尝试：
    - 仅使用中文关键词，例如 `"仓颉 异常处理"`；
    - 或在英文关键词后追加简短中文说明，例如 `"HTTP client 客户端示例"`。

### 结果展示要求

- **必须**将终端中返回的 L1 检索结果原样贴出给用户查看，至少包含：
  - 命中的知识点标题/路径；
  - 关键代码片段与上下文说明；
  - 与当前问题高度相关的排错步骤（如果存在）。
- 不要只给出你自行总结后的结论，用户需要能看到原始检索内容。

### 严禁行为（Stop and Wait）

在执行完 L1 RAG 检索命令后，你必须：

1. 等待终端返回完整结果；
2. 仔细阅读检索结果，根据结果质量判断后续动作；
3. 在未见到终端返回内容之前，**严禁**凭空编写仓颉代码或推断 API 用法。

> 若终端输出中明显提示 `NO_RAG_RESULT` 或结果内容与当前问题基本无关，应立即交由 `cangjie-doc-search-guide` 执行 L3 官方文档检索兜底。

### 自我评估（Self-Reflection）

对每一次 L1 检索，你都应进行如下评估：

- 若检索结果中已经包含：
  - 明确的 API 名称与调用方式；
  - 完整或接近完整的代码示例；
  - 与当前错误堆栈高度匹配的排错建议；
  
  则可以在理解这些内容后，直接基于结果进行代码修改或补全。

- 若检索结果：
  - 内容较泛，难以直接落到代码层；
  - 或与当前工程环境/版本明显不符；
  - 或完全不相关；
  
  则应立刻转入 `cangjie-doc-search-guide`，通过 L3 官方文档进行更精确的“考古级”检索。

## 与其它 Skills 的协作关系

- 与 `cangjie-kernel`：
  - `cangjie-kernel` = 规则化、结构化的仓颉基础知识索引；
  - `cangjie-vec-retriever-guide` = 当规则索引命中失败/不足时的“语义放大镜”；
  - 两者互补，不应互相替代。
- 与 `cangjie-doc-search-guide`：
  - 当前 Skill 负责 **L1 语义检索层**；
  - 当 L1 无 法提供足够支撑时，交由 `cangjie-doc-search-guide` 进入 **L3 官方文档精读层**；
  - 你在调用 L3 前，应尽量将 L1 中得到的关键词/模块名带入官方文档检索流程，以缩小搜索范围。
